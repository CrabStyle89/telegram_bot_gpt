# Telegram GPT Multi-Bot (aiogram 3)

Мультифункціональний Телеграм-бот на базі сучасного асинхронного фреймворку **aiogram 3** та штучного інтелекту **ChatGPT (OpenAI API)**. Бот має модульну архітектуру (роутери), використовує систему станів (FSM) для збереження контексту розмови та підтримує кілька інтерактивних режимів.

## Основні функції та режими

* ** ChatGPT Чат (`/gpt`)** — класичний режим спілкування з асистентом ChatGPT у вільному форматі.
* ** Розмова з особистістю (`/talk`)** — духовний сеанс та діалог з відомими людьми (Курт Кобейн, Альберт Ейнштейн, Ілон Маск). Бот завантажує унікальні системні промпти та веде розмову від імені персонажа, зберігаючи історію реплік.
* ** Інтерактивний Квіз (`/quiz`)** — перевірка знань на різні теми (Python, математика, біологія). Бот динамічно генерує питання через ШІ, автоматично перевіряє відповіді користувача та веде живий рахунок правильних відповідей.
* ** Розумний Перекладач (`/translate`)** — швидкий переклад текстів на англійську, німецьку, іспанську або польську мови без зайвого "спаму" та вступних слів від ШІ, з можливістю гарячої зміни мови через інлайн-кнопки.
* ** Гік-Рекомендації (`/recommend`)** — персональний підбір фільмів, книг, музики чи аніме за жанром та настроєм. Режим має **динамічний Blacklist** — якщо натиснути "Не подобається", поточний твір додається до чорного списку у пам'ять FSM і більше ніколи не рекомендується в поточній сесії.
* ** Випадковий Факт (`/random`)** — видача цікавих наукових або життєвих фактів.

---

## Технологічний стек

* **Мова програмування:** Python 3.11+
* **Фреймворк бота:** [aiogram 3.x](https://github.com/aiogram/aiogram) (Asynchronous Telegram Bot API)
* **Штучний інтелект:** OpenAI API (Модель ChatGPT)
* **Керування станами:** FSMContext (MemoryStorage)
* **Конфігурація:** python-dotenv (для безпечного збереження токенів)

## Структура проекту

```text
telegram_bot_gpt/
├── .env                  # Файл конфігурації (токени та ключі)
├── bot.py                # Головний файл запуску бота
├── util.py               # Допоміжні утиліти, класи FSM-станів, сервіс чату
├── credentials.py        # Файл з токенами
├── gpt.py                # Логіка і підключення АІ
├── handlers/             # Папка з модулями обробки повідомлень
│   ├── __init__.py       # Збірний роутер проекту (абсолютні імпорти)
│   ├── start.py          # Головне меню та базові команди
│   ├── random.py         # Хендлер випадкових фактів
│   ├── chat_gpt.py       # Режим вільного чату та розмови з зірками (/talk)
│   ├── quiz.py           # Інтерактивний квіз із рахунком балів
│   ├── translate.py      # Розумний перекладач
│   └── recommend.py      # Рекомендації з функцією Blacklist
│   └── talk.py           # Розмова з зірковою особистістю
└── resources/            # Ресурси бота (картинки, тексти промптів)
    ├── images/           # Зображення для кожного режиму (.jpg)
    ├── messages/         # Текстові повідомлення для інтерфейсу
    └── prompts/          # Системні промпти для ChatGPT (*.txt)

⚙️ Інсталяція та запуск
1. Клонування репозиторію

Звантаж проект на свій комп'ютер:
Bash

git clone [https://github.com/CrabStyle89/telegram_bot_gpt](https://github.com/your_username/telegram_bot_gpt.git)
cd telegram_bot_gpt

2. Налаштування віртуального оточення

Створи та активуй віртуальне середовище Python:
Bash

# Для Windows
python -m venv venv
venv\Scripts\activate

# Для macOS / Linux
python3 -m venv venv
source venv/bin/activate

3. Встановлення залежностей

Встанови необхідні бібліотеки:
Bash

pip install aiogram python-dotenv openai

4. Налаштування змінних оточення

Створи в корені проекту файл з назвою .env і додай туди свої токени:
Code snippet

TELEGRAM_TOKEN=твої_токен_від_bot_father
OPENAI_API_KEY=твій_секретний_ключ_openai

    ⚠️ Важливо: Ніколи не завантажуйте файл .env на GitHub! Переконайтеся, що він доданий у ваш файл .gitignore.

5. Запуск бота

Запусти проект через термінал або середовище розробки (наприклад, PyCharm):
Bash

python bot.py

💡 Особливості архітектури, враховані в проекті

    Захист від конфліктів кнопок: Усі Callback-кнопки розділені за унікальними текстовими префіксами (theme_, lang_, rec_cat_), що запобігає випадковому перехопленню подій дефолтним роутером.

    Порядок Роутерів: У файлі handlers/__init__.py підключення під-роутерів відбувається суворо перед дефолтним fallback_router, що забезпечує правильну чергу обробки повідомлень.

    Ізоляція введення тексту: Під час перегляду результатів (наприклад, у режимі рекомендацій) увімкнений FSM-стан блокує випадкове текстове введення користувача, змушуючи його користуватися кнопками, або дозволяє вийти через команду /start.
